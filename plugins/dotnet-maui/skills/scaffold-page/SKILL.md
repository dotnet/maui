# Scaffold a .NET MAUI ContentPage

Use this skill whenever you need to add a new page (screen) to a .NET MAUI app.
It produces a XAML file, a code-behind file, and the registration code needed to
navigate to the page.

---

## Step 1 ŌĆö Determine the Namespace

The namespace must match the project's root namespace plus the folder path.

| Project root namespace | Target folder | Resulting namespace |
|---|---|---|
| `MyApp` | `Views/` | `MyApp.Views` |
| `MyApp` | `Pages/` | `MyApp.Pages` |
| `MyApp` | project root | `MyApp` |

> Look at the `<RootNamespace>` in the `.csproj` file. If it is not set, it
> defaults to the project (assembly) name.

---

## Step 2 ŌĆö Create the XAML File

File name: **`{PageName}Page.xaml`** (e.g. `SettingsPage.xaml`).
Place it in the chosen folder (typically `Views/` or `Pages/`).

### Minimal Template (no ViewModel)

```xml
<?xml version="1.0" encoding="utf-8" ?>
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
             xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
             x:Class="MyApp.Views.SettingsPage"
             Title="Settings">

    <VerticalStackLayout Padding="16" Spacing="12">
        <Label Text="Settings go here"
               FontSize="18" />
    </VerticalStackLayout>

</ContentPage>
```

### Template with ViewModel Binding

When the page has an associated ViewModel, add the `xmlns:viewmodels` namespace
and `x:DataType` so the XAML compiler can verify bindings at build time:

```xml
<?xml version="1.0" encoding="utf-8" ?>
<ContentPage xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
             xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
             xmlns:viewmodels="clr-namespace:MyApp.ViewModels"
             x:Class="MyApp.Views.SettingsPage"
             x:DataType="viewmodels:SettingsViewModel"
             Title="{Binding Title}">

    <VerticalStackLayout Padding="16" Spacing="12">
        <Label Text="{Binding WelcomeMessage}"
               FontSize="18" />
        <Button Text="Save"
                Command="{Binding SaveCommand}" />
    </VerticalStackLayout>

</ContentPage>
```

### Adding the .NET MAUI Community Toolkit xmlns

If the page uses Toolkit converters, behaviors, or controls, add:

```xml
xmlns:toolkit="http://schemas.microsoft.com/dotnet/2022/maui/toolkit"
```

## Step 3 ŌĆö Create the Code-Behind File

File name: **`{PageName}Page.xaml.cs`** ŌĆö must sit next to the `.xaml` file.

### Without ViewModel

```csharp
namespace MyApp.Views;

public partial class SettingsPage : ContentPage
{
    public SettingsPage()
    {
        InitializeComponent();
    }
}
```

### With ViewModel (Constructor Injection)

```csharp
namespace MyApp.Views;

public partial class SettingsPage : ContentPage
{
    public SettingsPage(SettingsViewModel viewModel)
    {
        InitializeComponent();
        BindingContext = viewModel;
    }
}
```

> `InitializeComponent()` is generated by the XAML compiler from the `.xaml` file.
> It must be called before any code that references XAML-defined elements.

---

## Step 4 ŌĆö Register for Dependency Injection

In **`MauiProgram.cs`**, register both the page and its ViewModel so the DI
container can resolve them:

```csharp
// Pages
builder.Services.AddTransient<SettingsPage>();

// ViewModels
builder.Services.AddTransient<SettingsViewModel>();
```

Use `AddTransient` for pages and ViewModels so each navigation creates a fresh
instance. Use `AddSingleton` only when the page must preserve state across
navigations.

---

## Step 5 ŌĆö Register a Shell Route

For programmatic navigation, register a route in **`AppShell.xaml.cs`** (or
`MauiProgram.cs`):

```csharp
Routing.RegisterRoute(nameof(SettingsPage), typeof(SettingsPage));
```

Navigate to the page:

```csharp
await Shell.Current.GoToAsync(nameof(SettingsPage));
```

Alternatively, declare the page directly in **`AppShell.xaml`** as a tab or
flyout item:

```xml
<ShellContent Title="Settings"
              ContentTemplate="{DataTemplate views:SettingsPage}"
              Route="settings" />
```

When using `ContentTemplate`, Shell defers page creation until the user
navigates to it, which improves startup performance.

---

## File Naming Conventions

| File | Purpose |
|---|---|
| `SettingsPage.xaml` | XAML layout and UI declaration |
| `SettingsPage.xaml.cs` | Code-behind (partial class, constructor, event handlers) |
| `SettingsViewModel.cs` | ViewModel with bindable properties and commands |

- Always suffix page classes with **`Page`** (e.g. `SettingsPage`, `DetailPage`).
- Keep ViewModel classes in a separate `ViewModels/` folder.
- The `.xaml` and `.xaml.cs` files **must** share the same base name and reside
  in the same directory for the build system to pair them.

## Quick Checklist

1. **Namespace** matches `<RootNamespace>` + folder path.
2. **`x:Class`** in the XAML matches the fully qualified class name in the code-behind.
3. **`x:DataType`** is set when using compiled bindings with a ViewModel.
4. **`InitializeComponent()`** is the first call in the constructor.
5. **Page and ViewModel are registered** in `MauiProgram.cs` with `AddTransient`.
6. **Shell route is registered** if the page is reached via programmatic navigation.
7. **`.xaml` and `.xaml.cs`** share the same name and directory.
